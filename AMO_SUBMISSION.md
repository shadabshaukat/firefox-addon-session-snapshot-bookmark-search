# Mozilla Add-ons Publishing Guide

This project is structured as a publishable Firefox WebExtension. The package ZIP generated by `tools/package.py` is suitable for upload to Mozilla Add-ons (AMO).

## 1. Before first upload

1. Choose the final public add-on name.
2. Decide whether to keep the stable extension ID in `manifest.json`:
   - Current ID: `{4c06105c-65c9-4be4-87f7-cb793a16e43b}`
   - Do not change this after the first public release, or Firefox update continuity will break.
3. Review `PRIVACY_POLICY.md` and replace the copyright holder in `LICENSE` if needed.
4. Prepare screenshots for the AMO listing after loading the extension in Firefox.

## 2. Validate locally

Node/npm is not required for packaging. Run:

```bash
python3 tools/make_icons.py
python3 tools/validate.py
python3 tools/package.py
```

The uploadable package is created under `dist/`.

## 3. Upload to AMO

1. Create or log in to an AMO developer account: <https://addons.mozilla.org/developers/>
2. Choose **Submit a New Add-on**.
3. Select **On this site** for public distribution.
4. Upload `dist/session-snapshots-bookmark-search-0.1.0.zip`.
5. Use the permission justifications from `PERMISSIONS.md`.
6. The manifest already declares `browser_specific_settings.gecko.data_collection_permissions.required` as `["none"]`.
7. Use `PRIVACY_POLICY.md` as the privacy policy text or hosted privacy policy content.
8. Suggested category: **Productivity**.
9. Submit for Mozilla review and signing.

## 4. Suggested AMO listing copy

See `LISTING.md` for a short summary, full description, tags, and reviewer notes.

## 5. Release updates

For every update:

1. Update `manifest.json` version.
2. Update `package.json` version to match.
3. Run `python3 tools/validate.py`.
4. Run `python3 tools/package.py`.
5. Upload the new ZIP to AMO.
